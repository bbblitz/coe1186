1 Three main views

The main window contains three panels: a top ribbon that stores functions related files, windows, and display; a left pane that gives you an overview of the rail, and a right pane that shows you detailed information and allows you to schedual and dispatch trains, and control switches.

1.1 Top ribbon

The top ribbon stores functions related to files, windows, and displays. From left to right, the drop down menues on the ribbon are:
File | Edit | Window | Help

1.1.1 File

The file drop down allows you to exit the CTC window

1.1.2 Edit

The edit drop down allows you to:

  • copy a section of text to your clipboard with the copy button,
  • to copy to your clipboard and delete the selected text with the cut button,
  • or to paste the text in the currently selected text edit area with the paste button.
1.1.3 Window

The window drop down allows you to open any of the information views in their own window, for more information on each information view, see section 1.3

1.1.4 Help

The help drop down contains two options,

  • About will list the authors and name of the software
  • Manual will open this page in your default browser

1.2 Left pane

The left pane shows a diagram of the track

1.2.1 Track Symbols

The track is depicted with a 2 pixel line, additionally, tracks with special designation use different symbols:

  • Track designated Switch will show it's activated rail normally, and any non-active rails in a faded color.
  • Track designated Station will have additional 4px lines on either side of the track, the same color as the track.
  • Track designated Crossing will have additional 2px blue lines on either side of the track.
1.2.2 Train Symbols

The trains are depicted in this pane as rectangles. Rectangles with an arrow inside are engines, the arrow points the same direction that the engine is faceing.

1.2.3 Colors

The 2px track can assume the following colors for the following reasons:

  • Normal(green)
  • Broken Rail(Gray)
  • Track Circuit Failure(Red)
  • Power Failure(Yellow)

If the section of track is experienceing more than one failure at a time, it will be striped with the colors of each failure in regular intervals

1.3 Right pane

The right pane shows more information and contains the controls for elemenets selected in the left pane.

1.3.1 Train schedule pane

The train schedule pane allows you to schedual setting the authority of a train at a future time. For dispatching a train immediately, see section 1.3.4

1.3.2 Train line visibility pane

The train line visibility pane allows you so set which lines are visible in the left pane. A checked box indicates that the line is currently visible. To toggle the visibility of a line, click on the checkbox next to it's label.

1.3.3 Train details pane

The train details pane displays details about the currently selected train. To select a train, simply click on it in the left pane. This pane will show you the train's name, throughput of the track it is currently on, current number of passengers, current speed, and current authority.

1.3.4 Train dispatch pane

The train dispatch pane allows you do dispatch a selected train to any station immediately. Once a train has been selected in the left pane, the train's name will be displayed at the top of the dispatch pane, you can now enter the station the train should be dispatched to, and press set

1.3.5 Message dialog pane

The message dialog pane will display messages about the transit line, the messages may include rail failures, track switches, and trains executing their schedual. This is neither a complete or exhaustive list. More recent messages are displayed at the bottom, messages that have scrolled past the top of the pane can be viewed by scrolling up.

1.3.6 Track switch pane

The track switch pane will display the state of a selected switch on the line, and will allow the user to change the direction the switch is faceing. The current state of the switch will be displayed in the color of the track(see Section 1.2.3), and will display all non-active possible states in a faded color. Clicking on an inactive track will switch the track view, but will not activate the switch until the confrim button is pressed.



2 Common functions

This sections describes breifly how to do some of the common functions of the CTC, without bothering with detail explaniations like in section 1

2.1 Show/hide train lines

All train lines are visible by default the first time the program starts. The names of each line can be seen the the Train line visibility pane on the right hand side of the window (This pane can also be opened in a seperate window by selecting Line Vis. under the Window drop down at the top of the window). You can toggle the hidden state of a line by clicking the checkbox next to the line's name. It is not possible to set a line to half, quarter, or any other visibility than "on" and "off". Hideing a line will also hide any trains currently on the line.

2.2 viewing a train's information

To view a train's information, select it on the left hand side of the window, it's details will appear in the right hand side of the window, in the Train details pane(This pane can also be opened in a seperate window by selecting Details under the Window drop down at the top of the window). This pane will show the train's name, the throughput of the section of track it is on, the number of passengers currently abord, the train's current speed, and the train's authority.

2.3 Manually dispatching a train

To manuall dispatch the train, use the Train Dispatch Pane located on the right side of the window.(This pane can also be opened in a seperate window by selecting Dispatch under the Window drop down at the top of the window). To dispatch a train, select it in the left side of the window by clicking on it. You can varify it is selected by seeing if it's information is visible in the Train Details Pane. To dispatch, enter the station to dispatch it to, and press the set button at the bottom of the dispatch pane.

2.4 Schedualing trains

To schedual a train to be dispatched in the future, use the Train Schedual Pane, located on the right hand side of the window.(This pane can also be opened in a seperate window by selecting Schedual under the Window drop down at the top of the window). To use the pane, slect the train you want to schedual from the left side of the window. You can varify it is selected by seeing if it's information is visible in the Train Details Pane. It's current schedual will appear in the train schedual pane, it there is nothing currently in the schedual, it will be blank. To add items to the schedual, click the add button at the bottom of the pane, a popup will apear, prompting your for the time of the new schudel item, and the station to dispatch the train to at that time. Once you have entered the time and station, press the confirm button. If you ever accidentally enter an incorrect schedual, you can change the schedual before it executes by selecting it in the table, and pressing the edit or delete buttons in the shedualing pane.

2.5 Moveing switches

To change the direction a switch is faceing, use the track switch pane, located on the right side of the window.(This pane can also be opened in a seperate window by selecting Schedual under the Window drop down at the top of the window). To use this pane, select a section of track designated as a switch section (see Section 1.2.1), the switch will display in the switch pane, with current direction the switch is faceing in the primary color of the track, and directions the switch can face, but is not faceing in a faded color. To move the switch, select one of the faded color tracks, and the view in the switch pane will change, but the switch will not be flipped yet. To finish moveing the switch, press the Confirm button at the bottom of the switch pane.



3 Errors and troubleshooting

If possible, errors will be shown in a popup dialog, if not, they will be displayed to the console.

User errors

Errors caused by the user

3.1.1 I am not a train

This error occures when the user has selected a section of track, and attempts to do operations that apply to trains, such as schedualing, or dispatching.



To resolve, select a train.

3.1.2 I am not a track

This error occures when the user has selected a train, and attempts to do operations that apply to tracks, such as moveing switches.



To resolve, select a section of track.

Software errors

These may require parts of the software to be rewritten


These are serious software errors that may force portion of the software to be re-written

Errors in track file parseing

Error 1xx - Parser errors

Error 101 - The parser hit the end of the file while in a comment block, place a */ before the end of the file to close the block

Error 102 - The parse hit the end of the file while createing a line, place LINE END on the last line of the file to designate the end of the line

Error 103 - A line in the track file does not use the correct syntax for a segment of track

Error 104 - A block is supposed to be connected to a track segment that was never created, be sure the block head/tail is correct
